Karpenter는 적절한 노드를 적시에 사용하여 Kubernetes 인프라를 단순화합니다.

Karpenter는 클러스터의 애플리케이션을 처리하는 데 적합한 컴퓨팅 리소스만 자동으로 시작합니다. Kubernetes 클러스터를 위하여, 빠르고 간단한 컴퓨팅 프로비저닝을 통해 클라우드를 최대한 활용할 수 있도록 설계되었습니다.

{{< /blocks/lead >}} {{< blocks/section color="primary" >}} {{< blocks/feature title="애플리케이션 가용성 향상" icon="fa-globe" feature_class="feature" >}}

Karpenter는 애플리케이션 로드, 스케줄링 및 리소스 요구사항의 변화에 신속하고 자동으로 대응하여 사용 가능한 다양한 컴퓨팅 리소스 용량에 새로운 워크로드를 배치합니다.

{{< /blocks/feature >}} {{< blocks/feature title="운영 오버헤드 최소화" icon="fa-wrench" feature_class="feature" >}}

Karpenter는 쉽게 사용자 정의할 수 있는 선언적인 단일 Provisioner 리소스에 있는 일련의 독창적인 기본값으로 운영 오버헤드를 최소화합니다. 추가 구성이 필요하지 않습니다!

{{< /blocks/feature >}} {{< /blocks/section >}} {{< blocks/section color="dark" type="section" >}}

How It Works

Karpenter는 스케쥴 되지 않은 포드의 총 리소스 요청을 지켜보며, 스케줄링 지연 시간과 인프라 비용을 최소화 하기 위해 노드들의 시작 및 종료 결정을 내립니다.

{{< /blocks/section >}} {{< blocks/lead color="grey-100" type="section" >}}

Karpenter는 오픈소스 소프트웨어 입니다.

Karpenter는 Permissive 한 Apache License 2.0에 따라 라이선스가 부여됩니다. Karpenter는 모든 주요 클라우드 공급자와 사내 환경을 포함한 환경에서 실행되는 모든 Kubernetes 클러스터와 함께 작동하도록 설계되었습니다.

기능에 대한 아이디어가 있거나 더 잘 작동할 수 있는 기능으 찾으셨습니까? GitHub issue를 만들어서 우리에게 알려주세요.
